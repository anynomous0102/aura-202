The modern AI ecosystem is heavily fragmented. Users must switch between dozens of platforms, managing multiple API keys, just to compare outputs. Our project proposes a unified frontend aggregator backed by a dynamic context-switching protocol engine.

🗂️

Horizontal Aggregation

Instead of linear chat histories, outputs from multiple models are presented in horizontally aligned, independent cards. This UI paradigm allows instant cross-referencing and comparison of AI "opinions" without scrolling.

🔑

Keyless Access

Users do not need to provide their own API keys for OpenAI, Anthropic, or local models. The backend utilizes decentralized IDs (ANP) and intelligent pooling to route requests securely on behalf of the user.

⚡

Context Switching

Rather than forcing a single protocol like JSON-RPC, the system analyzes the prompt and dynamically selects the best transport layer (MCP, ACP, Agora) to guarantee low latency and high accuracy.

Phase 1

The 5 Core Protocols

To create a flawless amalgamation, we researched and integrated five leading alternatives and complementary protocols. Each serves a highly specialized function.

Model Context Protocol (MCP)

Enterprise

Often called the "Gold Standard" for vertical integration. While A2A connects agents to each other, MCP connects a single agent to tools, databases, and local data with strict enterprise-grade security and audit trails.

Best for: Database queries, private local data.

Agent Comm. Protocol (ACP)

Lightweight

Developed by IBM Research, ACP is considered "better" for lightweight, local environments. It uses standard REST principles, meaning it doesn't require a heavy SDK and is highly compatible with legacy enterprise systems.

Best for: Fast RESTful routing, legacy systems.

Agent Network Protocol (ANP)

Decentralized

Advanced for decentralized or cross-company networks. It uses W3C Decentralized Identifiers (DIDs) to allow agents to find and trust each other across the open internet, transcending corporate firewalls.

Best for: Cross-platform agent trust.

Agora

Research/NLP

An advanced research protocol from the University of Oxford. It allows agents to autonomously negotiate their own communication rules using natural language, rather than following rigid JSON-RPC structures.

Best for: Unstructured data, autonomous agents.

Agent-User Interaction (AG-UI)

Frontend Layer

The absolute standard for our frontend presentation layer. It is specialized for capturing the output from MCP, ACP, etc., and streaming the agent's "thinking" and tool calls directly to our Horizontal Cards UI in real-time via WebSockets.

Best for: Low-latency WebSocket streaming to the browser.
Phase 2

Capability Analytics

Why switch contexts? Our research phase evaluated the protocols across multiple dimensions. No single protocol scored perfectly across all metrics, proving the necessity of an amalgamation engine.

Multi-Dimensional Strengths

Comparing feature focus across protocols.

Latency vs. Security Overhead

Lower latency is better. Higher security implies more overhead.

Key Finding:

While MCP provides the highest security for data access, its overhead creates latency. Conversely, ACP is lightning fast but lacks decentralized trust features found in ANP. The solution is not to pick one, but to analyze the prompt and route accordingly.

Phase 3

The Amalgamation Engine

This is the brain of the college project. When a prompt is submitted, the Engine performs semantic analysis to detect intent, constraints, and data requirements, immediately routing the request to the optimal protocol.

Context Switching Decision Matrix

"private file"		 Route -> MCP Requires strict audit trails and local data access. MCP provisions the secure tunnel to the vector database.
"summarize fast"
"calculate"		 Route -> ACP Needs raw speed. ACP strips overhead, utilizing lightweight REST endpoints directly to the inference engine.
